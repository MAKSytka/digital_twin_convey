# Сценарии появления коробок

Поддерживаемый генератор:

```text
ros2 run singulator_sim singulation_row_spawner
```

Он создаёт одну поперечную волну за вызов и использует Gazebo-сервис `create_multiple`, чтобы коробки одной волны появились почти одновременно.

## Вероятности числа коробок

| Число коробок в волне | Вероятность |
|---:|---:|
| 4 | 40% |
| 3 | 30% |
| 2 | 15% |
| 1 | 15% |

Среднее число коробок в волне равно 2,95.

## Паттерны

### Четыре коробки

```text
four_standard
```

Четыре небольшие коробки занимают все условные полосы.

### Три коробки

```text
three_standard
```

Выбираются три из четырёх полос.

### Две коробки

Подпаттерны выбираются с вероятностью 50/50.

Неодинаковая пара:

```text
two_asymmetric_<left_profile>_<right_profile>
```

Профили: `long`, `compact`, `wide`.

Крупная и маленькая:

```text
large_small_<profile>_large_side_<sign>
```

Профили крупной коробки: `long`, `wide`, `skewed`.

### Одна коробка

```text
one_large_long_diagonal
one_large_wide_oblique
one_large_medium_skewed
```

Одиночная коробка крупная и заметно повёрнута.

## Расчёт периода волн

Спавнер сравнивает два периода:

1. период, необходимый для заданной средней интенсивности;
2. период освобождения зоны спавна с учётом максимальной длины, разброса X и безопасного зазора.

Используется больший из них. Это уменьшает вероятность появления новой волны внутри предыдущей.

При стандартных параметрах:

```text
infeed_speed_mps = 2.0
target_rate_boxes_per_sec = 4.0
maximum_box_length_m = 0.40
x_jitter_m = 0.08
safety_gap_m = 0.10
```

период определяется целевой интенсивностью и составляет около `2.95 / 4 = 0.7375 с` симуляционного времени.

## Параметры запуска

```bash
ros2 run singulator_sim singulation_row_spawner --ros-args \
  -p use_sim_time:=true \
  -p world_name:=matrix_14x4_stream \
  -p spawn_x:=-3.60 \
  -p x_jitter_m:=0.08 \
  -p belt_top_z:=0.08 \
  -p infeed_speed_mps:=2.0 \
  -p target_rate_boxes_per_sec:=4.0 \
  -p maximum_box_length_m:=0.40 \
  -p safety_gap_m:=0.10 \
  -p seed:=42
```

Одинаковый `seed` воспроизводит последовательность выбора параметров при одинаковом порядке вызовов.

## Имена моделей

```text
box_<session>_w<wave_id>_l<position>
```

Очистка удаляет модели, имя которых начинается с `box_`, при достижении `X >= 3.60 м` или падении ниже `Z <= -0.50 м`.
