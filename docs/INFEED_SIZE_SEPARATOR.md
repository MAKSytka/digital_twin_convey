# Демонстрация входного размерного сепаратора

## Назначение

Стенд показывает предварительный механический отсев мелких товаров перед
основной матрицей сингуляризации. Входная линия, верхний выход и нижний выход
имеют ширину 2,5 м. Скорость всех лент по умолчанию равна 2 м/с.

После роликового экрана поток разделяется:

- товар, уверенно перекрывающий отверстия экрана, продолжает движение по
  верхнему трёхметровому конвейеру;
- мелкий или узкий товар проваливается на нижнюю приёмную ленту и затем
  движется по отдельному трёхметровому конвейеру.

Коробки автоматически удаляются после достижения конца соответствующей ветви.

## Геометрия и скорость минироликов

Экран состоит из 11 физических поперечных валов с шагом 120 мм. Каждый вал
вращается вокруг оси Y через одно `revolute`-соединение и один
`gz::sim::systems::JointController`.

На одном валу размещены 25 контактных дисков:

- шаг дисков по ширине: 100 мм;
- упрощённая цилиндрическая коллизия: радиус 25 мм;
- контактный диаметр: 50 мм;
- толщина диска: 30 мм;
- чистое отверстие по X: `120 - 50 = 70 мм`;
- чистое отверстие по Y: `100 - 30 = 70 мм`.

Визуальный зуб имеет наружный размер 64 мм, но физическое взаимодействие
рассчитывается по цилиндрической коллизии радиусом 25 мм. Поэтому скорость
контактной поверхности определяется именно по 25 мм, а не по визуальному
радиусу зуба.

Для целевой линейной скорости 2 м/с:

```text
omega = v / r = 2 / 0.025 = 80 rad/s
n = omega * 60 / (2*pi) = 763.9 rpm
```

Контроллер вычисляет угловую скорость из параметров
`screen_surface_speed_mps` и `shaft_collision_radius_m` и выводит в логах
радиус, диаметр, rad/s и rpm.

## Правило размерного класса

Для основания `L × W`, повёрнутого на угол `yaw`, вычисляются две проекции:

```text
projection_x = |L cos(yaw)| + |W sin(yaw)|
projection_y = |L sin(yaw)| + |W cos(yaw)|
```

Ожидаемый нижний маршрут назначается по правилу:

```text
min(projection_x, projection_y) < 0.070 м  -> LOWER
иначе                                      -> UPPER
```

Так учитывается не только короткий товар, но и длинная узкая упаковка, которая
опасна для 20-миллиметровых зазоров основной матрицы. Для чистой демонстрации
верхний класс имеет минимальную проекцию не менее 90 мм. Область около 70 мм
следует отдельно калибровать.

## Профили генерируемых коробок

Вероятность нижнего класса по умолчанию равна 20%. Внутри классов используются
разные профили, а не один общий диапазон.

### Нижняя ветвь

- `micro_parcel`: компактный минимальный товар от 35×15×10 мм;
- `long_narrow`: длинная коробка 250–400 мм с шириной 25–55 мм;
- `flat_strip`: плоская длинная упаковка толщиной 10–30 мм;
- `tall_slender`: высокая узкая коробка с малой опорной шириной;
- `near_cutoff`: товар с малой стороной 55–69 мм для проверки границы.

### Верхняя ветвь

- `medium_carton`: обычная средняя коробка;
- `large_carton`: крупная коробка до 400×320×280 мм;
- `long_parcel`: длинная, но достаточно широкая упаковка;
- `flat_panel`: плоская панель или коробка-книга;
- `tall_carton`: высокая коробка с устойчивым основанием;
- `square_carton`: близкая к квадратной упаковка с большими углами поворота.

Профиль добавляется в имя модели и в лог спавнера. Статистика показывает число
успешно созданных товаров каждого профиля.

## Масса и инерция

Масса больше не определяется только объёмом с минимальным ограничением 10 г.
Используется сумма:

```text
mass = bulk_density * volume + cardboard_areal_density * surface_area
```

Затем применяется минимальная масса профиля. Например:

- `micro_parcel`: не менее 0,08 кг;
- `long_narrow`: не менее 0,20 кг;
- `tall_slender`: не менее 0,25 кг;
- средняя коробка: не менее 0,35 кг;
- крупная коробка: не менее 1,00 кг.

Максимальная масса сохранена на уровне 5 кг. Моменты инерции автоматически
пересчитываются как для прямоугольного параллелепипеда по фактической массе и
габаритам.

## Подавление нереалистичных ударов

Для коробок в этом сценарии применяются следующие параметры:

```text
spawn_clearance_m                     = 0.002
box_restitution                       = 0.02
bounce_capture_velocity_mps           = 0.35
linear_velocity_decay                 = 0.05
angular_velocity_decay                = 0.30
contact_max_correcting_velocity_mps   = 0.05
```

Изменения относительно прежней модели:

1. Коробка появляется в 2 мм над лентой вместо 8 мм, поэтому начальный удар
   слабее.
2. Коэффициент восстановления равен 0,02; при скорости удара ниже 0,35 м/с
   эффективная упругость обнуляется.
3. Угловое затухание подавляет длительное вращение лёгких коробок после
   случайного касания.
4. Ограничена скорость коррекции проникновения контакта, чтобы численный
   решатель не выбрасывал маленький товар резким импульсом.
5. Для ODE и Bullet добавлены контактные stiffness/damping параметры; общий
   `bounce` и `velocity_decay` остаются независимыми от выбранного движка.
6. `allow_auto_disable=true` разрешает физическому движку усыплять успокоившиеся
   тела и уменьшает дрожание.

Параметры доступны как launch-аргументы, поэтому их можно калибровать без
редактирования Python.

## Входящий поток

Центр масс каждой коробки выбирается из десяти фиксированных координат Y на
ширине 2,5 м. Размер, высота, масса и угол коробки меняются случайно.

Параметры по умолчанию:

```text
spawn_mode                  = continuous
target_rate_boxes_per_sec   = 4.0
small_item_probability      = 0.20
cutoff_m                    = 0.070
conveyor_speed_mps          = 2.0
screen_surface_speed_mps    = 2.0
seed                        = 42
```

Искусственная постоянная сила к коробкам не прикладывается. Продольное движение
возникает от контакта с лентами и вращающимися дисками.

## Запуск

```bash
cd ~/singulator_digital_twin
source /opt/ros/jazzy/setup.bash
colcon build --symlink-install
source install/setup.bash
ros2 launch singulator_bringup infeed_size_separator_demo.launch.py
```

Пример конечного теста:

```bash
ros2 launch singulator_bringup infeed_size_separator_demo.launch.py \
  spawn_mode:=finite \
  maximum_items:=200 \
  target_rate_boxes_per_sec:=4.0 \
  small_item_probability:=0.20
```

Более вязкий режим для диагностики ударов:

```bash
ros2 launch singulator_bringup infeed_size_separator_demo.launch.py \
  box_restitution:=0.0 \
  angular_velocity_decay:=0.45 \
  contact_max_correcting_velocity_mps:=0.03
```

## Логи

`separator_demo_controller` выводит скорость лент, контактный радиус и диаметр
дисков, линейную скорость поверхности, угловую скорость в rad/s и rpm.

`separator_demo_spawner` выводит профиль, размеры, массу, угол, обе опорные
проекции, выбранный spot, ожидаемый маршрут, фактическую частоту и распределение
профилей.

`separator_demo_cleanup` выводит фактический маршрут, ошибки классификации,
число товаров на ветвях, время прохождения и результаты удаления.

## Статическая проверка

```bash
python3 tools/validate_separator_demo.py

python3 -m py_compile \
  src/singulator_control/singulator_control/separator_demo_controller.py \
  src/singulator_sim/singulator_sim/separator_demo_spawner.py \
  src/singulator_sim/singulator_sim/separator_demo_cleanup.py \
  src/singulator_bringup/launch/infeed_size_separator_demo.launch.py \
  tools/validate_separator_demo.py
```

После сборки необходимо проверить в Gazebo направление вращения валов,
стабильность переходов, долю ошибок около 70 мм и real-time factor при 4
товарах/с.
