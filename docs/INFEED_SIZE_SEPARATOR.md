# Демонстрация входного размерного сепаратора

## Назначение

Стенд показывает предварительный механический отсев мелких и узких товаров
перед основной матрицей сингуляризации. Входная линия, верхний выход и нижний
выход имеют ширину 2,5 м. Скорость движущихся поверхностей по умолчанию равна
2 м/с.

После роликового экрана поток разделяется:

- товар, уверенно перекрывающий отверстия, продолжает движение по верхнему
  трёхметровому конвейеру;
- мелкий или узкий товар проваливается на нижнюю приёмную ленту и движется по
  отдельному трёхметровому конвейеру.

Коробки удаляются после достижения конца соответствующей ветви.

## Геометрия и скорость минироликов

Экран состоит из 11 поперечных валов с шагом 120 мм. Каждый вал является одной
физической сборкой с одним `revolute`-соединением и одним
`gz::sim::systems::JointController`.

На валу размещены 25 контактных дисков:

- шаг дисков по ширине — 100 мм;
- радиус упрощённой цилиндрической коллизии — 25 мм;
- контактный диаметр — 50 мм;
- толщина диска — 30 мм;
- чистое отверстие по X — `120 - 50 = 70 мм`;
- чистое отверстие по Y — `100 - 30 = 70 мм`.

Визуальный зуб имеет наружный размер 64 мм, но физическая скорость поверхности
рассчитывается по контактному радиусу 25 мм:

```text
omega = v / r = 2 / 0.025 = 80 rad/s
n = omega * 60 / (2*pi) = 763.9 rpm
```

Серый поперечный вал больше не является только текстурой. Для него добавлена
цилиндрическая коллизия радиусом 8 мм и длиной 2,480 м, совпадающая с
визуальной геометрией. Поэтому коробка не может проходить сквозь видимый вал.
При этом товар всё ещё может падать через реальные промежутки между соседними
валами.

## Переходы на выходные конвейеры

Причиной заторов был вертикальный передний торец выходной ленты. При зазоре
9 мм коробка могла потерять опору на последнем ролике и удариться в этот торец.

Оба перехода изменены одинаково:

- горизонтальный зазор уменьшен до 1 мм;
- принимающая лента расположена на 4 мм ниже предыдущей поверхности;
- скорость принимающей ленты остаётся 2 м/с.

Верхний переход идёт от последнего ролика к `accepted_belt`. Нижний переход
идёт от `reject_transfer_belt` к `reject_belt`. Небольшой переход вниз
предпочтительнее ступени вверх: передняя грань принимающего конвейера больше не
оказывается на траектории нижней поверхности коробки.

## Правило размерного класса

Для основания `L × W`, повёрнутого на угол `yaw`, вычисляются две проекции:

```text
projection_x = |L cos(yaw)| + |W sin(yaw)|
projection_y = |L sin(yaw)| + |W cos(yaw)|
```

Ожидаемый маршрут:

```text
min(projection_x, projection_y) < 0.070 м  -> LOWER
иначе                                      -> UPPER
```

Так длинная коробка шириной, например, 40 мм не будет ошибочно принята только
из-за большой длины.

## Профили генерируемых коробок

Вероятность класса `LOWER` по умолчанию теперь равна 50%. Это позволяет быстро
наблюдать работу нижней ветви и деспавнера при коротком тесте.

### Нижняя ветвь

- `micro_parcel` — компактный минимальный товар от 35×15×10 мм;
- `long_narrow` — длина 250–400 мм, ширина 25–55 мм;
- `flat_strip` — плоская длинная упаковка толщиной 10–30 мм;
- `tall_slender` — высокая узкая коробка с малой опорной шириной;
- `near_cutoff` — малая сторона 55–69 мм для проверки границы.

### Верхняя ветвь

- `medium_carton` — обычная средняя коробка;
- `large_carton` — крупная коробка до 400×320×280 мм;
- `long_parcel` — длинная, но достаточно широкая упаковка;
- `flat_panel` — плоская панель или коробка-книга;
- `tall_carton` — высокая коробка с устойчивым основанием;
- `square_carton` — близкая к квадратной упаковка.

## Масса и контактная динамика

Масса оценивается как сумма массы содержимого и картонной оболочки:

```text
mass = bulk_density * volume
     + cardboard_areal_density * surface_area
```

Затем применяется минимальная масса профиля. Для коробок также используются:

```text
spawn_clearance_m                     = 0.002
box_restitution                       = 0.02
bounce_capture_velocity_mps           = 0.35
linear_velocity_decay                 = 0.05
angular_velocity_decay                = 0.30
contact_max_correcting_velocity_mps   = 0.05
```

Это уменьшает отскоки, длительное вращение и численные импульсы при исправлении
контактного проникновения.

## Деспавнер

Текущие имена моделей содержат профиль:

```text
box_separator_123_n0000001_exp_lower_long_narrow_spot04
box_separator_123_n0000002_exp_upper_large_carton_spot07
```

Старый шаблон деспавнера ожидал `exp_lower_spot04` и игнорировал новые имена.
Шаблон исправлен и принимает как имена с профилем, так и старый формат.

Дополнительная устойчивость:

- монитор `/world/<world>/pose/info` автоматически перезапускается после
  неожиданного завершения `gz topic`;
- удаление повторяется до трёх раз;
- в логах показываются причины повторов и ответы сервиса;
- статистика содержит `monitor_restarts` и `remove_failures`.

## Запуск

После изменения SDF-моделей рекомендуется очистить установленные копии
затронутых пакетов:

```bash
cd ~/singulator_digital_twin
source /opt/ros/jazzy/setup.bash

rm -rf \
  build/singulator_description \
  build/singulator_gazebo \
  build/singulator_bringup \
  build/singulator_sim

rm -rf \
  install/singulator_description \
  install/singulator_gazebo \
  install/singulator_bringup \
  install/singulator_sim

colcon build --symlink-install
source install/setup.bash

ros2 launch singulator_bringup \
  infeed_size_separator_demo.launch.py
```

Конечный тест:

```bash
ros2 launch singulator_bringup \
  infeed_size_separator_demo.launch.py \
  spawn_mode:=finite \
  maximum_items:=200 \
  target_rate_boxes_per_sec:=4.0 \
  small_item_probability:=0.50
```

## Проверка

```bash
python3 tools/validate_separator_demo.py

python3 -m py_compile \
  src/singulator_control/singulator_control/separator_demo_controller.py \
  src/singulator_sim/singulator_sim/separator_demo_spawner.py \
  src/singulator_sim/singulator_sim/separator_demo_cleanup.py \
  src/singulator_bringup/launch/infeed_size_separator_demo.launch.py \
  tools/validate_separator_demo.py
```

После запуска следует проверить:

1. коробки больше не пересекают видимый серый вал;
2. верхний и нижний стыки не образуют устойчивый затор;
3. в статистике растёт `removed`;
4. приблизительно половина созданных коробок имеет ожидаемый маршрут `LOWER`;
5. real-time factor остаётся приемлемым при 4 товарах/с.
