# Демонстрация входного размерного сепаратора

## Почему отдельный мир

Первый этап реализован отдельным миром Gazebo. Это изолирует механику
размерного отсева от матрицы 18×4, алгоритма V7 и машинного зрения. После
проверки геометрии порога модуль можно без переработки встроить перед основной
матрицей: весь сепаратор оформлен как модель
`model://infeed_size_separator`.

## Принцип работы

Модуль содержит четыре движущиеся поверхности:

1. входную ленту;
2. верхний роликовый экран;
3. верхнюю ленту для принятого товара;
4. нижнюю ленту для мелкого товара.

Верхний экран состоит из 11 поперечных звёздчатых валов с шагом 120 мм.
Ширина физической контактной полосы одного вала равна 32 мм, поэтому открытый
промежуток между соседними опорами составляет 88 мм. Визуальная форма вала
собрана из трёх пересекающихся профилей, а поступательное воздействие на товар
реализовано через `TrackController`. Это устойчивее полнотелой динамической
модели десятков вращающихся зубчатых дисков и подходит для первичной
верификации потока.

Номинальный порог нельзя трактовать как точный габарит коробки: прохождение
зависит от угла, проекции основания, центра масс и контакта сразу с несколькими
валами. В демонстрации заложены уверенные классы:

- проекция до 70 мм — ожидаемый сброс на нижнюю ленту;
- проекция от 105 мм — ожидаемый проход по верхнему уровню;
- диапазон 70–105 мм — переходная зона, которую нужно калибровать.

## Запуск

```bash
cd ~/singulator_digital_twin
source /opt/ros/jazzy/setup.bash
colcon build --symlink-install
source install/setup.bash

ros2 launch singulator_bringup infeed_size_separator_demo.launch.py
```

Параметры повторяемости:

```bash
ros2 launch singulator_bringup infeed_size_separator_demo.launch.py \
  spawn_period_s:=1.8 \
  cycles:=4
```

Последовательно создаются крупные, мелкие, средние, плоские мелкие и
диагональные товары. В логах указывается ожидаемый маршрут `UPPER` или `LOWER`.
Зелёные и синие товары должны оставаться на верхнем уровне, оранжевые и
красные — падать на нижнюю ленту. Узел `separator_demo_cleanup` отслеживает
положение только коробок этого сценария и удаляет их после схода с верхнего
или нижнего выходного конвейера. Поэтому после завершения заданного числа
циклов мир остаётся чистым и его можно перезапустить без ручной очистки.

## Статическая проверка

```bash
python3 tools/validate_separator_demo.py
```

Проверка подтверждает корректность XML, число валов, размер открытого
промежутка, наличие четырёх управляющих топиков, launch-файла и ROS entry
points. Она не заменяет запуск Gazebo.

## Дальнейшая интеграция

После калибровки отдельного мира модуль следует поставить перед текущим
входом матрицы 18×4. Верхний выход подключается к основной матрице, нижний —
к отдельной мелкоячеистой зоне сингуляризации. При интеграции потребуется
расширить поле зрения, сдвинуть точку спавна и добавить две независимые зоны
трекинга.
