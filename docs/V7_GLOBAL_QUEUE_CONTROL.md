# V7: неизменяемая глобальная очередь и прямое управление зазорами

## Причина перехода от V6

V6 фиксировал ранги только внутри отдельных кластеров и продолжал использовать
абсолютные временные слоты выхода. При потере raw-ID, временном объединении двух
контуров или создании нового кластера один и тот же физический товар мог снова
попасть в другую локальную группу. Кроме того, общая конвейерная ячейка
назначалась одному «владельцу», хотя физически одновременно воздействовала на
несколько коробок.

V7 удаляет обе причины:

- нет повторного формирования кластеров в середине матрицы;
- нет календаря абсолютных exit slots;
- нет выбора единственного владельца общей ячейки.

## Поток управления

```text
BoxObservationArray
  -> логические товары поверх raw track ID
  -> обнаружение объединённых контуров
  -> захват входной волны
  -> неизменяемая глобальная очередь
  -> прямой регулятор зазоров соседних товаров
  -> желаемая скорость каждого товара
  -> cell-aware оптимизация распределения на 56 ячеек
  -> MatrixCommand
```

## 1. Неизменяемая глобальная очередь

На входе используется виртуальная линия `entry_gate_x`. Товары, обнаруженные в
одном окне `entry_capture_window_s`, считаются одной входной волной. Они один раз
сортируются по убыванию координаты `Y` и добавляются в конец общей очереди.

После добавления относительный порядок уже находящихся в очереди товаров не
изменяется. Текущая координата `X` не используется для повторной сортировки.

## 2. Прямой adjacent-gap controller

Для соседних товаров вычисляется фактический продольный зазор:

```text
clearance = x_leader - half_length_leader
            - x_follower - half_length_follower
```

Пока зазор меньше целевого, регулятор требует положительную разность скоростей:

```text
v_leader - v_follower =
    gap_gain * gap_deficit
    + relative_velocity_gain * closing_speed
```

Для поперечной группы из четырёх коробок начальный профиль при полном дефиците
может достигать:

```text
3.00, 2.33, 1.67, 1.00 м/с
```

После получения требуемых зазоров все товары автоматически возвращаются к
`transport_speed_mps = 2.50 м/с`. Никакого отдельного состояния повторной
сингуляризации нет.

### Deadline recovery перед выходом

На последних `deadline_separation_distance_m` метрах обычный регулятор
дополняется расчётом по оставшемуся времени. Для пары с дефицитом он требует
минимальную разность скоростей, необходимую для достижения
`target_gap + deadline_gap_margin_m` до выхода из матрицы. Это предотвращает
ситуацию, когда небольшой зазор обнаружен слишком поздно и товар попадает в
горлышко несингуляризованным. Команда всё равно ограничена
`maximum_relative_speed_mps`; если общая опора физически не позволяет создать
разность скоростей, пара остаётся `uncontrollable`.

## 3. Логическая идентичность поверх raw-ID зрения

Контроллер хранит собственный `uid`. Raw-ID машинного зрения является только
текущей привязкой к наблюдению.

При изменении raw-ID новое наблюдение сопоставляется с существующим логическим
товаром по прогнозируемой координате, поперечному смещению и габаритам. Поэтому
смена ID не создаёт новый элемент очереди автоматически.

## 4. Объединённые контуры и ghost tracks

Если одна увеличенная рамка покрывает прогнозируемые позиции двух и более
логических товаров, она трактуется как merged observation.

В этом случае:

- большая рамка не создаёт новый одиночный товар;
- исходные `uid` остаются в глобальной очереди;
- их координаты прогнозируются до `merged_track_timeout_s`;
- разделяющие команды продолжают формироваться;
- после появления отдельных рамок raw-ID заново привязываются к прежним `uid`.

## 5. Общие ячейки

V7 не выбирает одного владельца общей поверхности. Команды ячеек вычисляются
совместно: минимизируется ошибка между целевой и фактической, усреднённой по
площади контакта, скоростью каждого товара. Это лучше независимого усреднения
запросов по ячейкам: при частично общей опоре частные ячейки сохраняют градиент
скоростей и продолжают раздвигать товары.

```text
weight = 1 + allocation_urgency_gain * urgency
```

Небольшой `allocation_idle_regularization` удерживает неиспользуемые ячейки
около `idle_speed_mps`, а `allocation_iterations` задаёт число быстрых итераций
ограниченного решения. Это уменьшает переключение команд и отражает физический
факт, что одна лента одновременно воздействует на все лежащие на ней коробки.

Если две соседние коробки имеют почти одинаковые векторы контакта с ячейками,
пара помечается как `uncontrollable`: в текущей позиции матрица физически не
может сообщить им разные скорости. Контроллер ждёт появления различной опоры,
не переключая произвольно владельца ячейки.

## 6. Коррекция yaw

Yaw-коррекция отключается для плотной пары, если её зазор меньше
`dense_pair_yaw_gap_m`. Сначала создаётся продольный интервал, и только затем
разрешается корректировать ориентацию. Также yaw-коррекция отключена возле
роликового горлышка, чтобы не передавать коробке поздний вращательный импульс.

## 7. Независимый финальный ряд

Перед роликовым горлышком добавлен ряд из четырёх независимых лент
`separator_c00…separator_c03` длиной 360 мм. Он получает те же целевые скорости
товаров, что и основная матрица, но распределяет их только по своему контакту.
Поэтому пара, имевшая одинаковую опору на последних ячейках матрицы, получает
ещё один независимый участок для формирования зазора перед входом в горлышко.
Ряд управляется топиками `/singulator/separator/cXX/cmd_vel`; его команды также
ограничены диапазоном `1.00…3.00 м/с` и ограничением ускорения контроллера.

## Диагностика

Лог `control_v7` содержит:

```text
queue
merged
reidentified
ghosts
orphans
inversions
unresolved_exit
deadline_boost
min_gap
shared_cells
uncontrollable
allocation_error
```

Главные критерии:

```text
inversions = 0
unresolved_exit = 0
```

`allocation_error` показывает, насколько эффективная скорость товара отличается
от его целевой скорости из-за общей опоры на ячейки.

## Проверка

```bash
python3 tools/test_v7_logic.py
./scripts/check_v7_control.sh
```

Порядок прогонов:

```text
2 ед/с -> 3 ед/с -> 4 ед/с
```

Для сравнения версий необходимо сохранять одинаковый `SEED` и отдельно считать
долю товаров с недостаточным зазором на выходе.
