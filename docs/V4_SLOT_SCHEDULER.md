# V4: зональное управление и планирование выходных слотов

## Причина замены V3

V3 одновременно использовал целевые позиции, межкластерное торможение и индивидуальный ограничитель продольного отставания. Эти механизмы могли выдавать противоречивые требования: заднюю коробку требовалось замедлить для разделения, а ограничитель отставания в тот же момент повышал её минимальную скорость.

V4 удаляет индивидуальную эталонную траекторию и строит управление вокруг одного критерия: времени пересечения выхода матрицы каждой коробкой.

## Физические пределы

- минимальная команда ячейки: `1.0 м/с`;
- максимальная команда: `3.0 м/с`;
- ограничение ускорения: `6.0 м/с²`;
- ограничение рывка TrackController: `30.0 м/с³`;
- скорость роликового горлышка: `2.5 м/с`;
- частота контроллера: `30 Гц`.

Минимальная скорость `1.0 м/с` не позволяет алгоритму удерживать коробку на одном ряду в течение нескольких циклов появления новых товаров. Разделение выполняется главным образом ускорением лидера и заранее назначенным временем выхода.

## Зоны матрицы

| Зона | Ряды | Базовое поведение |
|---|---:|---|
| Захват | `r00–r02` | Незанятые ячейки работают на `2.8 м/с`; кластер быстро входит в рабочую область |
| Разделение | `r03–r11` | Основная команда определяется выходным слотом; ранговый профиль используется как небольшой feed-forward |
| Стабилизация | `r12–r13` | Команда плавно сближается со скоростью роликов `2.5 м/с` |

Зоны задают базовое поле даже без наблюдаемого товара. Машинное зрение не рассчитывает всю динамику с нуля, а только назначает коробке кластер, ранг и принадлежащие ей ячейки.

## Кластеры и ранги

Новые товары объединяются в кластер по ограниченному диапазону координаты `X`. Транзитивное объединение запрещено: полный диапазон `max(X)-min(X)` кластера должен оставаться меньше `cluster_dx_enter_m`.

Ранги задаются один раз:

```text
rank 0 — наибольшая координата Y;
rank 1 — следующая координата Y;
...
```

После задержки `cluster_slot_lock_delay_s` ранги блокируются.

## Назначение выходных слотов

Для `rank 0` назначается время `T0`. Время следующей коробки определяется её габаритом, габаритом предыдущей коробки, требуемым зазором и скоростью выхода:

```text
Tk = T(k-1) + (half_length(k-1) + half_length(k) + target_gap) / roller_speed
```

При назначении проверяется достижимый интервал каждой коробки:

```text
earliest = now + distance / maximum_speed
latest   = now + distance / minimum_speed
```

Если общий набор слотов не помещается в эти интервалы, кластер помечается `overloaded`. Контроллер продолжает выдавать ограниченные команды, но не скрывает физическую недостижимость сценария.

## Команда скорости

Основная оценка:

```text
required_speed = distance_to_exit / time_to_slot
```

В центральной зоне она смешивается с небольшим ранговым профилем:

```text
rank 0 -> 3.0 м/с
...
последний rank -> 1.0 м/с
```

Профиль не заменяет обратную связь: он только начинает формировать порядок до накопления большой ошибки времени.

## Защита между кластерами

При недостаточном межкластерном расстоянии V4 сдвигает выходные времена текущего и всех следующих кластеров. Это отличается от V3: задержка становится частью общего расписания и не отменяется независимым регулятором.

При критически малом расстоянии дополнительно применяется мгновенный предел скорости заднего кластера.

## Диагностика

Лог `control_v4` содержит:

```text
boxes
clusters
slots
unresolved
overloaded
slot_error
slot_delays
intra_gap
inter_gap
conflict_cells
speed=[min,max]
```

Ключевые признаки:

- `overloaded > 0` — расписание физически недостижимо при диапазоне `1..3 м/с`;
- растущий `slot_error` — коробки не успевают следовать назначенным временам;
- частые `slot_delays` — входной поток догоняет предыдущие кластеры;
- отрицательный `intra_gap` — коробки перекрываются по продольной проекции;
- `inter_gap < 0.12 м` — активируется жёсткая защита между кластерами.

## Первый план испытаний

Для каждого запуска фиксировать `seed`, число прошедших товаров, число неразрешённых кластеров на выходе и максимальную ошибку слота.

```bash
TARGET_RATE_BOXES_PER_SEC=2.0 SEED=42 ./scripts/run_roller_demo.sh
TARGET_RATE_BOXES_PER_SEC=3.0 SEED=42 ./scripts/run_roller_demo.sh
TARGET_RATE_BOXES_PER_SEC=4.0 SEED=42 ./scripts/run_roller_demo.sh
```

После проверки одного seed повторить минимум для:

```text
1, 2, 3, 4, 5, 42, 100
```

Не следует одновременно изменять скорость роликов, целевой зазор и ранговый вес: иначе невозможно определить причину изменения результата.
