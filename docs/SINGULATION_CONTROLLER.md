# Замкнутый алгоритм сингуляризации

## Назначение первой версии

Контроллер превращает поперечные волны коробок в **одну продольную очередь**:
между продольными проекциями соседних товаров формируется заданный зазор, а
поворот коробок постепенно уменьшается. Финальное сведение координаты `Y` к
центру лучше выполнять механическим горлышком после матрицы. Экспериментальный
режим бокового руления предусмотрен параметром, но по умолчанию отключён.

## Контур управления

```text
/singulator/boxes
  -> оценка состояния и порядка товаров
  -> расчёт требуемой скорости каждой коробки
  -> пересечение габарита коробки с ячейками 14x4
  -> разрешение конфликтов и ограничение ускорения
  -> /singulator/matrix/command
```

Контроллер публикует 56 скоростей с частотой 20 Гц. Порядок row-major:
`index = row * 4 + col`.

## Логика

1. Новые товары получают постоянный номер очереди. Это не позволяет двум
   коробкам, идущим рядом, менять роли лидера и ведомого из-за шума координат.
2. Для каждой коробки считается продольная проекция с учётом yaw.
3. Лидер ускоряется. Скорость каждой следующей коробки определяется ошибкой
   расстояния до предшественника.
4. Если фактический зазор меньше аварийного, ведомая коробка получает
   минимальную скорость.
5. Скорость ячейки вычисляется по площади пересечения с прямоугольником товара.
6. Разность скоростей левой и правой части товара создаёт момент, уменьшающий
   yaw.
7. Если одна ячейка одновременно действует на два товара, выбирается меньшая
   скорость. Это безопасный режим: два товара на одном приводе независимо
   управляться не могут.
8. Изменение команды ограничено допустимым ускорением.

## Запуск

Не запускайте одновременно `uniform_matrix_controller` и
`singulation_controller`: оба публикуют один и тот же топик.

```bash
ros2 launch singulator_bringup matrix_stream.launch.py \
  start_perception:=true \
  start_spawner:=true \
  start_demo_controller:=false \
  start_singulation_controller:=true \
  infeed_speed_mps:=1.0 \
  outfeed_speed_mps:=2.0
```

Для первой настройки вход намеренно работает медленнее максимальной скорости
матрицы. Иначе при входе 2 м/с и ограничении ячеек 2 м/с у алгоритма нет запаса
для ускорения лидера.

## Проверка

```bash
./scripts/check_singulation.sh
```

В логе контроллера отображаются:

- число коробок в зоне управления;
- минимальный продольный зазор;
- число ячеек, одновременно занятых разными коробками;
- диапазон выданных скоростей.

## Начальные параметры

| Параметр | Значение | Смысл |
|---|---:|---|
| `base_speed_mps` | 1.20 | скорость незанятых ячеек |
| `leader_speed_mps` | 1.90 | скорость первой коробки |
| `target_gap_m` | 0.16 | целевой чистый зазор |
| `hard_gap_m` | 0.04 | аварийный порог |
| `gap_gain` | 1.80 | усиление регулятора расстояния |
| `yaw_gain` | 1.10 | усиление коррекции угла |
| `maximum_yaw_delta_mps` | 0.55 | предел разности скоростей под коробкой |
| `observation_timeout_s` | 0.65 | переход к безопасной равномерной скорости |

## Ограничения MVP

- Это детерминированный регулятор, а не глобальный оптимизатор.
- При одновременном нахождении нескольких товаров на одной ячейке управление
  ими физически связано.
- Стабильность порядка зависит от стабильных ID машинного зрения.
- Обычная матрица с движением лент только вдоль `X` хорошо управляет интервалом
  и углом, но ограниченно управляет чистым поперечным смещением.
- После настройки MVP следующий шаг — прогноз на 1–2 секунды и MPC/поисковая
  оптимизация 56 скоростей с функцией стоимости по зазорам, yaw и конфликтам.
