# Руководство по настройке алгоритма

## Цель настройки

Алгоритм должен одновременно обеспечивать:

1. только положительное движение товара вдоль `+X`;
2. формирование продольного зазора;
3. ограниченное отставание от траектории движения со скоростью `2 м/с`;
4. отсутствие столкновений между соседними волнами;
5. постепенную коррекцию угла;
6. устойчивый вход в роликовое горлышко.

## Рекомендуемый порядок

### Этап 1. Один товар

```bash
TARGET_RATE_BOXES_PER_SEC=0.5 ./scripts/run_roller_demo.sh
```

Проверить:

- товар проходит стык матрица → ролики без удара;
- координата X монотонно растёт;
- скорость команд не ниже `minimum_speed_mps`;
- ролики направляют товар к центру.

### Этап 2. Две коробки одной волны

```bash
TARGET_RATE_BOXES_PER_SEC=1.0 ./scripts/run_roller_demo.sh
```

Настраивать сначала:

- `target_gap_m`;
- `gap_gain`;
- `leader_speed_mps`;
- `yaw_gain`.

### Этап 3. Несколько волн

```bash
TARGET_RATE_BOXES_PER_SEC=2.0 ./scripts/run_roller_demo.sh
```

Следить за:

```text
min_gap
max_lag
conflict_cells
speed=[min,max]
```

### Этап 4. Номинальный поток

```bash
TARGET_RATE_BOXES_PER_SEC=3.0 ./scripts/run_roller_demo.sh
TARGET_RATE_BOXES_PER_SEC=4.0 ./scripts/run_roller_demo.sh
```

## Как менять параметры

### Коробки недостаточно разделяются

По очереди:

1. увеличить `leader_speed_mps` до `2.9–3.0`;
2. увеличить `gap_gain` небольшими шагами, например `2.2 → 2.5`;
3. увеличить `prediction_horizon_s`, например `0.18 → 0.22`;
4. увеличить `longitudinal_control_margin_m`, например `0.16 → 0.20`.

Не уменьшать `minimum_speed_mps` до нуля и не возвращать реверс.

### Коробки слишком сильно отстают

1. уменьшить `maximum_longitudinal_lag_m`: `0.30 → 0.25 → 0.20`;
2. увеличить `lag_guard_horizon_s`;
3. увеличить `lag_recovery_gain`;
4. уменьшить `target_gap_m`, если зазор избыточен.

### Коробки сталкиваются между волнами

1. проверить, что нет отрицательных команд;
2. проверить единственного издателя `/singulator/matrix/command`;
3. уменьшить интенсивность спавна;
4. увеличить `safety_gap_m` спавнера;
5. увеличить прогноз и упреждение алгоритма.

### Коробки резко вращаются

1. уменьшить `yaw_gain`;
2. ограничить разность скоростей соседних колонок;
3. уменьшить максимальное изменение команды за такт;
4. проверить `mu2` и фактический контакт товара с несколькими колонками.

### Коробки ударяются о ролики

Проверить:

- верх матрицы `Z=0.080 м`;
- верх collision роликов `Z=0.080 м`;
- переходную пластину;
- отсутствие выступающей визуальной или collision-геометрии;
- установленную копию модели в `install/...`, а не только файл в `src/...`.

## Методика фиксации результатов

Для каждого опыта сохранять:

```text
seed
интенсивность потока
набор параметров алгоритма
real-time factor
минимальный наблюдаемый зазор
максимальное отставание
число столкновений
число успешно прошедших товаров
```

Изменять за один опыт желательно один или два параметра. Иначе невозможно понять причину улучшения или ухудшения.
