# Чек-лист передачи второму программисту

## Автор проекта

- [ ] Репозиторий открывается и не содержит `build/`, `install/`, `log/`.
- [ ] `python3 tools/validate_project.py` завершается успешно.
- [ ] Создан стабильный тег `v0.1-handoff`.
- [ ] Зафиксированы Ubuntu, ROS 2 и Gazebo.
- [ ] Объяснён порядок 56 скоростей.
- [ ] Указано, что `/singulator/boxes` пока не публикуется.
- [ ] Указано, что `MatrixState.actual_speed_mps` пока является заглушкой.
- [ ] Крупные бинарники вынесены во внешнее хранилище.
- [ ] Ссылки и SHA-256 записаны в `external_artifacts/README.md`.

## Второй программист

- [ ] Клонировал репозиторий в чистую папку.
- [ ] Выполнил `./scripts/setup_dependencies.sh`.
- [ ] Выполнил `./scripts/build.sh`.
- [ ] Запустил `./scripts/run_demo.sh`.
- [ ] Увидел движение коробок по всей матрице.
- [ ] Запустил `./scripts/run_scenario.sh`.
- [ ] Опубликовал собственный `MatrixCommand`.
- [ ] Проверил `r00_c00` и `r13_c03`.
- [ ] Согласовал формат входных данных о коробках.
- [ ] Создал отдельную Git-ветку для алгоритма.

## Совместная приёмка

- [ ] Один участник демонстрирует запуск с нуля по README.
- [ ] Второй участник не использует команды из личной переписки.
- [ ] После клонирования не требуется копировать файлы вручную.
- [ ] Все параметры запуска находятся в launch-файле, скриптах или YAML.
- [ ] Известные ограничения отражены в README.
- [ ] Есть короткая видеозапись стабильной версии.
