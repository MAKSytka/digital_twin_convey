# V3: улучшенное зрение и поэтапная кластерная сингуляризация

## Причина изменения

При потоке `4 товара/с` предыдущая версия теряла качество по трём причинам:

1. морфологическая обработка маски могла соединять соседние коробки;
2. небольшие или низкоконтрастные коробки иногда отбрасывались до публикации `/singulator/boxes`;
3. контроллер пытался одновременно ускорять, замедлять и выравнивать коробки по всей матрице без отдельной зоны захвата входящей волны.

## Изменения зрения

- `background_threshold`: `25 -> 18`;
- `inner_erode_px`: `6 -> 2`;
- минимальная площадь контура: `30 -> 18 px`;
- закрытие маски: `7x7 -> 3x3`;
- постоянная дилатация отключена;
- добавлено watershed-разделение соединённых компонентов;
- трекер сильнее штрафует поперечную перестановку ID;
- число допустимых пропусков трека увеличено до `12` кадров.

В отладочном изображении выводятся:

```text
detections=<число распознанных коробок>
split_components=<число компонентов, которые detector разделил на части>
```

## Стадии управления

### 1. Boost

Первые три ряда действуют как зона запуска кластера. Пока кластер целиком не прошёл эту зону, все его товары получают одинаковую команду до `3.0 м/с`.

Это создаёт запас расстояния относительно следующей волны. Внутри boost-зоны не выполняется дифференциальная коррекция yaw, чтобы кластер не разрывался раньше, чем контроллер зафиксировал состав и ранги.

### 2. Meter

Если впереди недостаточно свободного пространства, новый кластер не запускается на полной скорости. Он получает ограниченную скорость `entry_meter_speed_mps`, пока межкластерный зазор не станет безопасным.

### 3. Transition

Один переходный ряд снижает общую скорость кластера с `3.0` до `2.55 м/с` перед началом активного разделения.

### 4. Separate

Товары одного кластера получают постоянные ранги:

```text
rank 0 -> наибольшая координата Y
rank 1 -> следующая координата Y
...
```

Для рангов строятся движущиеся целевые позиции вдоль X. Первый товар должен идти впереди, каждый следующий — за ним с учётом половин габаритов и `target_gap_m`.

### 5. Finalizer

На последних трёх рядах увеличивается коэффициент позиционного управления, лидер может получать `3.0 м/с`, а влияние коррекции угла уменьшается. Приоритет — закончить продольное разделение до роликового горлышка.

### 6. Resolved

После удержания требуемых зазоров в течение `cluster_resolution_hold_s` кластер считается разрешённым. Его целевые позиции продолжают двигаться с номинальной скоростью `2.0 м/с`.

## Диапазон скоростей

Рабочий диапазон команд:

```text
0.10 ... 3.00 м/с
```

Скорость `0.10 м/с` разрешается только пока сохраняется бюджет продольного отставания. Параметр `lag_guard_horizon_s=0.15` позволяет реально использовать низкую скорость после входного ускорения. При исчерпании бюджета нижняя граница команды автоматически повышается.

## Основные параметры

```text
minimum_speed_mps              = 0.10

entry_boost_rows               = 3

```

## Запуск

```bash
cd ~/singulator_digital_twin
./scripts/stop_roller_demo.sh 2>/dev/null || true
./scripts/build.sh
source install/setup.bash
TARGET_RATE_BOXES_PER_SEC=4.0 SEED=42 ./scripts/run_roller_demo.sh
```

Интерфейс зрения:

```bash
ros2 run rqt_image_view rqt_image_view /singulator/perception/debug_image
```

Диагностика:

```bash
./scripts/check_vision.sh
./scripts/check_singulation.sh
./scripts/check_positive_flow.sh
ros2 topic hz /singulator/boxes
ros2 topic echo /singulator/matrix/command --once
```

Лог контроллера V3 содержит:

```text

unresolved
boost
meter
finalizer
intra_gap
inter_gap
conflict_cells
max_lag
speed=[min,max]
```

## Методика испытаний

Для сравнения использовать одинаковые seed:

```bash
for seed in 1 2 3 4 5 42 100; do
  TARGET_RATE_BOXES_PER_SEC=4.0 SEED="$seed" ./scripts/run_roller_demo.sh
  # записать результаты, затем остановить процесс
  ./scripts/stop_roller_demo.sh
done
```

Фиксировать:

- число физических коробок в кадре;
- число опубликованных наблюдений;
- число `split_components`;
- количество смен ID;
- долю кластеров, разрешённых до finalizer;
- долю несингуляризированных кластеров на выходе;
- минимальный межкластерный зазор;
- максимальное продольное отставание.

## Ограничения

Watershed не может гарантированно разделить две коробки, если в изображении между ними нет ни зазора, ни различимой границы. Для промышленного варианта потребуются instance segmentation либо использование предыдущих треков как маркеров разделения.

V3 прошёл статическую проверку синтаксиса Python. Его поведение необходимо проверить в Gazebo на конкретной машине и затем настроить по записанным метрикам.
