# Аудит исходного архива

## Что было обнаружено

Исходный архив содержал рабочие исходники, но также включал:

- `build/`, `install/`, `log/`;
- `src_before_rtf_optimization/` и `scripts_before_rtf_optimization/`;
- файлы `*.before_*`, `*.backup`, `__pycache__`;
- несколько устаревших генераторов коробок;
- основной README, ориентированный на раннюю версию мира.

## Несоответствия, исправленные в handoff-версии

1. `matrix.yaml` описывал старую матрицу 5040×700 мм и другие зазоры, тогда как основной мир использует активную ячейку 360×175 мм, зазоры 20 мм и габарит 5,30×0,76 м.
2. Основной `matrix.launch.py` запускал не потоковый мир `matrix_14x4_stream.sdf`.
3. Старый bridge YAML содержал `/clock` и 56 ячеек, но не содержал входной и выходной конвейеры.
4. Ручные split-bridge скрипты не включали `/clock`, необходимый узлам с `use_sim_time=true`.
5. Не существовало единого launch-файла для полной линии.
6. README описывал `ground_truth_adapter`, которого в исходниках фактически нет.
7. `MatrixState.actual_speed_mps` является копией заданной команды, а не реальной одометрией.
8. Старые и текущие спавнеры были одновременно доступны как console scripts, что создавало неоднозначность.

## Что изменено

- удалены генерируемые и резервные файлы;
- основной спавнер оставлен один — `singulation_row_spawner`;
- добавлен `matrix_stream.launch.py`;
- bridge разбит на пять конфигураций;
- добавлены `/clock`, входной и выходной привод;
- добавлен ROS-узел `aux_conveyor_controller`;
- cleanup установлен как package executable;
- исправлена конфигурация геометрии;
- создан полный комплект документации и скриптов;
- добавлена статическая проверка проекта.

## Выполненные проверки

Успешно выполнены:

- разбор всех XML/SDF;
- разбор всех YAML;
- проверка наличия 56 уникальных ячеек;
- проверка управляющих топиков и bridge-покрытия;
- проверка значений `mu=0.8`, `mu2=0.2`;
- проверка структуры `MatrixCommand`;
- проверка синтаксиса Python и Bash;
- стресс-тест генераторов размеров коробок.

Полноценный запуск ROS 2 / Gazebo в среде подготовки архива не выполнялся, потому что в ней не установлены ROS 2 Jazzy и Gazebo. Финальная динамическая приёмка должна быть выполнена на Ubuntu 24.04 по `docs/HANDOFF_CHECKLIST.md`.
