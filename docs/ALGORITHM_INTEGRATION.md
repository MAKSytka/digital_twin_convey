# Интеграция алгоритма сингуляризации

## Минимальная ответственность алгоритма

На каждом шаге управления алгоритм должен сформировать 56 скоростей и опубликовать `MatrixCommand`.

```python
speeds = [0.0] * 56
speeds[row * 4 + col] = desired_speed_mps
```

Необходимо заполнять:

```text
rows = 14
cols = 4
target_speed_mps = 56 значений
```

## Рекомендуемая частота

Для первого подключения достаточно 20–50 Гц. TrackController имеет ограничение возраста команды `2.0 с`, поэтому единичная команда перестанет действовать после тайм-аута. Рабочий контроллер должен публиковать команды непрерывно.

## Этапы подключения

### Этап 1. Равномерный профиль

Все 56 скоростей одинаковы. Проверяется правильность размеров сообщения и направление движения.

### Этап 2. Управление одной строкой

Остальные строки движутся с базовой скоростью, одна строка получает разные скорости левой и правой половины.

```text
левые колонки c00, c01: base + delta
правые колонки c02, c03: base - delta
```

### Этап 3. Плавный профиль по нескольким строкам

Разность скоростей распределяется по соседним строкам, чтобы не создавать резкий скачок скорости и не опрокидывать лёгкие коробки.

### Этап 4. Замкнутый алгоритм

Контроллер получает координаты, размеры, угол и идентификатор коробки, вычисляет активные ячейки и формирует профиль скоростей.

## Ограничения команд

Физическая модель допускает `−2…2 м/с`, но рабочий алгоритм должен дополнительно ограничивать:

- максимальное изменение скорости между соседними тактами;
- разность скоростей соседних ячеек;
- длительность сильного вращательного воздействия;
- обратное движение, если оно не требуется сценарием.

Пример программного ограничения:

```python
def clamp(value: float, low: float = -2.0, high: float = 2.0) -> float:
    return max(low, min(high, value))
```

## Проверка публикации

```bash
ros2 topic hz /singulator/matrix/command
ros2 topic echo /singulator/matrix/command --once
ros2 topic echo /singulator/matrix/state --once
```

Ошибка размеров отображается в логе `matrix_command_fanout`:

```text
Matrix dimensions mismatch
Expected 56 speeds, got ...
```

## Интерфейс наблюдений

Подготовленный вход:

```text
/singulator/boxes
singulator_interfaces/msg/BoxObservationArray
```

Сейчас симулятор его не публикует. На первом этапе доступны три варианта:

1. алгоритм работает на заранее известных параметрах сценария;
2. отдельный тестовый узел публикует искусственные наблюдения;
3. создаётся адаптер из Gazebo Pose info в `BoxObservationArray`.

## Разделение ответственности

| Цифровой двойник | Алгоритм второго программиста |
|---|---|
| Gazebo-мир и физика | Выбор скоростей ячеек |
| Спавн и удаление коробок | Работа с наблюдениями коробок |
| ROS–Gazebo bridge | Планирование продольного интервала |
| Fan-out общей команды | Коррекция поперечного положения и угла |
| Тестовые контроллеры | Метрики алгоритмического качества |

Не следует изменять SDF или bridge ради реализации логики сингуляризации. Связь должна проходить через сообщения из `singulator_interfaces`.
