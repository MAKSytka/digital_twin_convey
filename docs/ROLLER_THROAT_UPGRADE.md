# Матрица ±3 м/с и роликовое сужающее горлышко

## Изменения физической модели

Новый мир `matrix_14x4_stream_v2.sdf` сохраняет геометрию матрицы 14×4, но
задаёт для входной ленты и всех 56 ячеек:

- скорость от `-3` до `+3 м/с`;
- ускорение от `-3` до `+3 м/с²`;
- `mu = 0.8` и `mu2 = 0.8`;
- ограничение рывка `±12 м/с³`.

Обычный выходной конвейер удалён. На его месте находится горлышко длиной
`0.80 м`, которое сужает рабочую ширину с `760` до `600 мм`.

## Геометрия роликов

Горлышко состоит из двух банков роликов. Оси левого банка ориентированы на
`+100°` (эквивалентно `-80°`) относительно направления движения, оси правого
банка — на `+80°`. Поэтому направление контактной скорости отклонено на
`+10°` и `-10°` от продольной оси: обе стороны одновременно перемещают товар
вперёд и к центру.

Визуально используются цилиндры. Для устойчивой работы TrackController в
Gazebo контакт каждого цилиндра аппроксимирован тонкой прямоугольной полосой,
совпадающей с его образующей. Это осознанная модель контактного поля, а не
симуляция отдельных подшипников и вращательных сочленений.

Боковые направляющие линейно сходятся от `±380` до `±300 мм`.

## Доработка алгоритма

Контроллер второй версии устраняет режим, при котором две коробки попадали на
одну ячейку и обе получали минимальную скорость:

1. Очередь определяется текущей координатой X, а постоянный sequence служит
   только разрешением ничьей для поперечной волны.
2. Скорость ведомого формируется относительно скорости лидера, ошибки зазора
   и относительной скорости сближения.
3. При аварийно малом зазоре лидер ускоряется, а ведомому разрешается
   ограниченный реверс до `-0.8 м/с`.
4. Положение прогнозируется на `0.18 с`, а зона управления расширяется на
   `0.16 м`, поэтому соседние ряды заранее готовят поверхность.
5. Конфликтная ячейка передаётся ближайшей прогнозируемой коробке, а не всегда
   получает минимум всех предложений.
6. Изменение скорости ограничено `3 м/с²`.

Физические границы контроллера составляют `-3…+3 м/с`, но полный реверс не
используется в штатном режиме: он оставлен как запас для последующих сценариев.

## Запуск

Старый контроллер внутри базового launch необходимо выключить, потому что
`matrix_stream_roller.launch.py` запускает улучшенный контроллер самостоятельно.

```bash
./scripts/stop_roller_demo.sh
./scripts/build.sh
source install/setup.bash
./scripts/run_roller_demo.sh
```

Ручной запуск:

```bash
ros2 launch singulator_bringup matrix_stream_roller.launch.py \
  start_perception:=true \
  start_spawner:=true \
  start_demo_controller:=false \
  start_singulation_controller:=false \
  start_cleanup:=true \
  infeed_speed_mps:=1.2 \
  outfeed_speed_mps:=0.0 \
  target_rate_boxes_per_sec:=2.0
```

## Камера наблюдателя

В новом мире начальная камера установлена сбоку: вся линия проходит поперёк
кадра, поэтому видны вход, матрица и роликовое горлышко. Для быстрого перехода
к горлышку:

```bash
./scripts/view_throat.sh
```

Команда вызывает сервис Gazebo `/gui/move_to` и фокусируется на модели
`roller_throat`.

## Проверка

```bash
./scripts/check_roller_upgrade.sh
```

На `/singulator/matrix/command` должен оставаться один издатель —
`singulation_controller`. `uniform_matrix_controller` в списке быть не должен.
