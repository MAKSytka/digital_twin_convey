# Совместная разработка

1. Не изменять `main` напрямую.
2. Создавать ветку `feature/...` или `fix/...`.
3. Перед коммитом запускать:

```bash
python3 tools/validate_project.py
git diff --check
```

4. После изменений ROS-пакетов выполнять чистую сборку или сборку затронутых пакетов.
5. Не хранить резервные копии пакетов внутри workspace.
6. Изменения `singulator_interfaces` согласовывать между всеми разработчиками.
7. В pull request описывать:
   - цель;
   - изменённые топики и параметры;
   - команды проверки;
   - известные ограничения.
